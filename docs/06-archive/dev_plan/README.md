# Historical Dev Plans

This directory stores dated development plans that were used during earlier
implementation rounds.

These files are useful for:

- implementation sequencing history
- gap analysis history
- validation scope history
- reasoning trace for major product decisions

They are not the current product contract.

## Usage Rule

Use these files when you need historical context or want to trace why a design
changed over time.

If a topic is already documented in the active docs under `docs/01` to
`docs/05`, prefer the active doc.

For future plan writing:

- New plan files must also be created in this directory.
- Do not recreate a root-level `dev_plan/`.
- Use the filename format:
  `YYYYMMDD_<short_description>.md`
- A plan document is an implementation-history artifact. After execution is
  complete, the current behavior should be reflected back into active docs under
  `docs/01` to `docs/05`.
