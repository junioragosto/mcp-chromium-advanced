# Archive

This directory stores historical planning and upgrade notes.

Archive files are historical records. They may describe superseded defaults,
older engine priorities, retired rollout sequencing, or design directions that
were later revised. Do not use archive plans as the current product contract.

These documents are useful for:

- implementation history
- reasoning trace for major redesigns
- upgrade sequencing and validation history

They are not the primary user-facing docs.

When a topic is already reflected in the current product docs under `docs/01` to
`docs/05`, prefer the current doc, not the archive plan.
